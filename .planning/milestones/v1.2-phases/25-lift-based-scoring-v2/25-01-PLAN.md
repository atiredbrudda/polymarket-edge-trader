<objective>
Replace the entire old scoring engine (40% WR + 25% concentration + 20% recency + 15% sample_size) with the backtest-validated lift-based formula: z(CLV) + z(ROI) + z(Sharpe) with equal weights. Create LiftScore ORM model, pure metric functions, rewire scoring pipeline, and rewire score/leaderboard CLI commands.

Purpose: The old formula's win_rate_component measures price preference, not skill. The backtest proved CLV+ROI+Sharpe outperforms all 84 tested alternatives (Spearman 0.54-0.67 vs ~0 for old formula). This plan replaces the engine so the pipeline produces actionable Q5 trader identification.

Output: Working `polymarket score` command that computes lift-based scores and `polymarket leaderboard` that displays Q1-Q5 ranked traders with CLV/ROI/Sharpe breakdown.
</objective>
