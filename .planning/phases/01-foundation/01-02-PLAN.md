**Rate Limiter (src/api/rate_limiter.py):**

Token bucket algorithm:
- `RateLimiter(max_per_second: int)` - configurable rate
- `acquire()` - blocks (sleeps) if rate limit would be exceeded, then allows request
- `acquire()` with no recent requests - returns immediately
- Thread-safe (use threading.Lock)

Test cases:
- `test_rate_limiter_allows_within_limit`: 3 requests within 1 second at rate=5 -> all proceed without delay
- `test_rate_limiter_blocks_over_limit`: Burst of max_per_second+1 requests -> last one delayed
- `test_rate_limiter_resets_after_window`: Fill bucket, wait for window, requests proceed again

**Pydantic Models (src/api/models.py):**

Define validation models matching Polymarket CLOB API responses:

`EventResponse`:
- `id: str` - event ID
- `title: str` - event title
- `slug: str` - URL slug
- `category: str` - Polymarket category
- `end_date: datetime | None` - end date (handle ISO string and Unix timestamp)
- `active: bool` - whether event is active
- `markets: list[MarketResponse]` - nested markets

`MarketResponse`:
- `condition_id: str` - unique market ID
- `question: str` - market question text
- `end_date_iso: str | None` - end date as ISO string
- `category: str` - inherited from parent event or direct
- `outcome: str | None` - resolution outcome
- `active: bool` - is market active
- `tokens: list[dict] | None` - token IDs and outcomes

`TradeResponse`:
- `id: str` - trade ID
- `market: str` - market/condition ID
- `trader: str` - wallet address (Field alias from "maker" or "taker")
- `side: str` - "BUY" or "SELL"
- `size: Decimal` - trade size
- `price: Decimal` - execution price
- `timestamp: datetime` - trade time (handle ISO and Unix timestamp)
- `asset_ticker: str | None` - token ticker

Use `model_config = ConfigDict(populate_by_name=True)` for alias support.
Add `@field_validator` for timestamp fields to handle both ISO and Unix formats.
Add `@field_validator` for price to ensure 0 < price < 1.

Test cases:
- `test_market_response_validates_complete_data`: Valid market dict -> MarketResponse
- `test_trade_response_decimal_precision`: Decimal values preserved, not float
- `test_timestamp_handles_iso_and_unix`: Both formats parse correctly

**API Client (src/api/client.py):**

`PolymarketClient` class:
- `__init__(settings: Settings | None = None)` - initializes py-clob-client ClobClient in read-only mode (no auth required for market data), creates RateLimiter from settings
- `get_events(active: bool = True) -> list[EventResponse]` - fetch events, paginate through all pages using `next_cursor`, validate with Pydantic. Uses Tenacity retry with exponential backoff on ConnectionError/TimeoutError.
- `get_markets(active: bool = True) -> list[MarketResponse]` - fetch simplified markets, paginate, validate
- `get_market_trades(condition_id: str, limit: int = 500) -> list[TradeResponse]` - fetch trades for a specific market (public endpoint). Paginate using cursor. This discovers traders by seeing who trades on a market.

IMPORTANT DESIGN NOTE (from Research open questions): py-clob-client's `get_trades()` returns only the authenticated user's trades. To discover traders and their activity, we must:
1. Fetch trades on specific MARKETS (public data) to discover trader addresses
2. Use market trade data to build trader profiles
3. The `get_market_trades` method fetches all trades on a given market, which reveals trader addresses

Each method must:
1. Call `self.rate_limiter.acquire()` before API request
2. Be decorated with `@retry(stop=stop_after_attempt(settings.retry_max_attempts), wait=wait_exponential(...), retry=retry_if_exception_type((ConnectionError, TimeoutError, httpx.HTTPError)))`
3. Log attempts via Loguru
4. Validate responses through Pydantic models
5. Handle pagination (Polymarket uses cursor-based pagination)

Test cases (use unittest.mock to mock ClobClient):
- `test_get_events_returns_validated_models`: Mock ClobClient response -> returns list[EventResponse]
- `test_get_markets_paginates`: Mock two pages of markets -> returns combined list
- `test_get_market_trades_returns_trades`: Mock trade data -> returns list[TradeResponse]
- `test_retry_on_connection_error`: Mock ConnectionError then success -> retries and succeeds
- `test_rate_limiter_called_before_request`: Verify rate_limiter.acquire() called before API call
