    Completely rewrite gamma_events table definition in create_core_tables():

    Current (wrong - JSON blob):
    ```python

        {
            "id": str,  # Primary key - hash
            "event_type": str,  # Type of event
            "market_condition_id": str,  # Associated market
            "data": str,  # JSON blob - WRONG

        },

        if_not_exists=True,
    )
    ```

    New (per GUIDE.md - normalized fields):
    ```python
    db.table("gamma_events").create(
        {
            "id": str,  # Primary key - hash
            "condition_id": str,  # Market condition ID (not market_condition_id)
            "question": str,  # Market question text
            "outcome": str,  # YES/NO/null after resolution
            "end_date": str,  # ISO timestamp - when market closes
            "tags": str,  # JSON array of tags from Gamma API
            "active": bool,  # Whether market is currently active
            "niche_slug": str,  # e.g., "esports" - set during ingest
            "created_at": str,  # ISO timestamp - when ingested
        },
        pk="id",
        if_not_exists=True,
    )
    ```

    Rationale per GUIDE.md:
    - condition_id: Direct link to markets table (resolve-outcomes joins on this)
    - question: Market question for display/debugging
    - outcome: Resolution outcome - resolve-outcomes reads this to update markets.outcome
    - end_date: Market close time - needed for CLV calculation in score
    - tags: JSON array from Gamma API (e.g., ["esports", "cs2", "IEM Katowice"])
    - active: Filter for active markets during discover phase
    - niche_slug: Scope entity extraction without JOIN to token_catalog
    - created_at: Audit trail for ingestion time

    Remove old columns: event_type, market_condition_id, data, timestamp

    Gap reason: JSON blob structure prevents resolve-outcomes from reading outcome field directly.
    Normalized columns allow simple SELECT outcome FROM gamma_events WHERE condition_id = ?
  </action>
