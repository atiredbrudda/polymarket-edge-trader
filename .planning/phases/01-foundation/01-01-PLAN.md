  <action>
Create `src/db/models.py` with SQLAlchemy 2.0 ORM models:

Use `from sqlalchemy.orm import DeclarativeBase` (SQLAlchemy 2.0 style, not `declarative_base()`).

Define `class Base(DeclarativeBase): pass`

**Market model** (`markets` table):
- `id: Mapped[int]` - primary key, autoincrement
- `condition_id: Mapped[str]` - Polymarket condition ID, unique
- `question: Mapped[str]` - market question text
- `category: Mapped[str]` - Polymarket category (stored as-is from API)
- `end_date: Mapped[datetime | None]` - market end date
- `outcome: Mapped[str | None]` - resolution outcome (null if unresolved)
- `active: Mapped[bool]` - whether market is currently active
- `tokens: Mapped[str | None]` - JSON string of token data (for downstream use)
- `created_at: Mapped[datetime]` - record creation timestamp
- `updated_at: Mapped[datetime]` - last update timestamp
- Indexes: `ix_market_category` on category, `ix_market_active` on active

**Trader model** (`traders` table):
- `id: Mapped[int]` - primary key
- `address: Mapped[str]` - wallet address, unique, indexed
- `first_seen: Mapped[datetime]` - when first discovered
- `last_active: Mapped[datetime]` - most recent activity
- `backfill_complete: Mapped[bool]` - whether 12-month history has been fetched
- `created_at: Mapped[datetime]` - record creation
- Index: unique on address

**Trade model** (`trades` table) - FULL DETAIL for target categories:
- `id: Mapped[int]` - primary key
- `market_id: Mapped[str]` - condition_id reference
- `trader_address: Mapped[str]` - trader wallet address
- `side: Mapped[str]` - "BUY" or "SELL"
- `size: Mapped[Decimal]` - trade size (use `Numeric(20, 6)` for Decimal precision)
- `price: Mapped[Decimal]` - execution price (use `Numeric(10, 6)`)
- `timestamp: Mapped[datetime]` - trade execution time
- `asset_ticker: Mapped[str | None]` - token ticker
- `trade_id: Mapped[str | None]` - Polymarket trade ID (for dedup), unique if not null
- `created_at: Mapped[datetime]` - record creation
- Composite indexes: `ix_trade_trader_timestamp` on (trader_address, timestamp), `ix_trade_category_timestamp` on (market_id, timestamp), `ix_trade_market_trader` on (market_id, trader_address)

**TraderCategorySummary model** (`trader_category_summaries` table) - AGGREGATE for non-target categories:
- `id: Mapped[int]` - primary key
- `trader_address: Mapped[str]` - wallet address
- `category: Mapped[str]` - Polymarket category
- `total_volume: Mapped[Decimal]` - sum of trade sizes (Numeric(20, 6))
- `trade_count: Mapped[int]` - number of trades
- `first_trade: Mapped[datetime]` - earliest trade in category
- `last_trade: Mapped[datetime]` - latest trade in category
- `updated_at: Mapped[datetime]` - last summary update
- Unique constraint on (trader_address, category)
- Index: `ix_summary_trader_category` on (trader_address, category) unique

Use `Mapped[]` type annotations with `mapped_column()` (SQLAlchemy 2.0 style). Use `Numeric` not `DECIMAL` for column type (cross-database compatible). Use `server_default` for timestamps where appropriate.

Create `src/db/session.py`:
- Define `create_engine_from_settings()` that:
  - Gets database_url from settings
  - Creates parent directory for SQLite file if it doesn't exist
  - Creates engine with `echo=False`
  - Executes `PRAGMA journal_mode=WAL` and `PRAGMA foreign_keys=ON` on connect (use `@event.listens_for(engine, "connect")`)
  - Returns engine
- Define `create_tables(engine)` that calls `Base.metadata.create_all(engine)`
- Define `get_session_factory(engine)` returning `sessionmaker(bind=engine)`
- Define context manager `get_session()` that yields a session, commits on success, rolls back on exception
- Define module-level `init_db()` that creates engine, creates tables, returns engine and session factory
