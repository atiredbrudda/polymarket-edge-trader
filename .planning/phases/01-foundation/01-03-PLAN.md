**CategoryFilter (src/pipeline/filters.py):**

`CategoryFilter` class:
- `__init__(detail_categories: list[str])` - takes list of categories requiring full detail storage
- `requires_detail(category: str) -> bool` - returns True if category matches a detail category (case-insensitive)
- `route_trades(trades: list[TradeWithCategory]) -> tuple[list, list]` - splits trade list into (detail_trades, summary_trades). Each trade needs a category attribute/field.

Where `TradeWithCategory` is a simple dataclass or Pydantic model:
```python
@dataclass
class TradeWithCategory:
    trade: TradeResponse  # from api/models.py
    category: str         # from the market's category
```

Test cases:
- `test_requires_detail_matching_category`: CategoryFilter(["eSports"]).requires_detail("eSports") -> True
- `test_requires_detail_case_insensitive`: CategoryFilter(["eSports"]).requires_detail("esports") -> True
- `test_requires_detail_non_matching`: CategoryFilter(["eSports"]).requires_detail("Politics") -> False
- `test_route_trades_splits_correctly`: Given 5 trades (3 eSports, 2 Politics), route_trades returns (3, 2) split
- `test_multiple_detail_categories`: CategoryFilter(["eSports", "Crypto"]).requires_detail("Crypto") -> True
- `test_empty_trade_list`: route_trades([]) -> ([], [])
- `test_all_detail_trades`: All trades in detail category -> all in detail list, none in summary

**Trade Aggregator (src/pipeline/aggregators.py):**

`aggregate_trades(trades: list[TradeResponse], trader_address: str, category: str) -> dict`:
- Takes a list of trades for a single trader in a single non-detail category
- Returns a dict compatible with TraderCategorySummary model:
  - `trader_address: str`
  - `category: str`
  - `total_volume: Decimal` - sum of all trade sizes
  - `trade_count: int` - number of trades
  - `first_trade: datetime` - earliest timestamp
  - `last_trade: datetime` - latest timestamp

`group_and_aggregate(trades: list[TradeWithCategory], trader_address: str) -> list[dict]`:
- Groups summary trades by category
- Calls aggregate_trades for each group
- Returns list of summary dicts

Test cases:
- `test_aggregate_single_trade`: One trade -> volume=size, count=1, first=last=timestamp
- `test_aggregate_multiple_trades`: 3 trades -> volume=sum of sizes, count=3, first=earliest, last=latest
- `test_aggregate_decimal_precision`: Uses Decimal, not float. Sum of Decimal(0.1) + Decimal(0.2) == Decimal(0.3)
- `test_group_and_aggregate_multiple_categories`: 5 trades across 2 categories -> 2 summary dicts
- `test_aggregate_preserves_trader_address`: Output dict contains correct trader_address
  </behavior>
