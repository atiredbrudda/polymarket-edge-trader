**Methods:**

`ingest_active_markets() -> int`:
1. Call `client.get_events(active=True)` to get all active events
2. For each event, extract markets
3. For each market, upsert into `markets` table (update if condition_id exists, insert if new)
4. Store: condition_id, question, category, end_date, outcome, active, tokens (as JSON string)
5. Batch insert/update in single transaction per 100 markets
6. Return count of markets processed
7. Log progress: "Ingested {N} active markets"

`discover_traders_from_market(condition_id: str) -> list[str]`:
1. Call `client.get_market_trades(condition_id)` to get all trades on a market
2. Extract unique trader addresses from trade data
3. For each new trader address, insert into `traders` table (skip if already exists)
4. Set `first_seen` and `last_active` timestamps
5. Return list of newly discovered trader addresses
6. Log: "Discovered {N} new traders from market {condition_id}"

`ingest_trader_history(trader_address: str) -> dict`:
This is the core per-trader ingestion that implements the locked decision:
1. Get all markets this trader has participated in by fetching trades from known markets
2. For each trade, look up market category from `markets` table
3. Create `TradeWithCategory` objects
4. Use `CategoryFilter.route_trades()` to split into detail and summary trades
5. **Detail trades** (eSports): Batch insert into `trades` table. Use trade_id for dedup (skip if trade_id already exists).
6. **Summary trades** (everything else): Call `group_and_aggregate()`, then upsert into `trader_category_summaries` table (update if trader+category exists, insert if new)
7. Mark trader as `backfill_complete=True`, update `last_active`
8. Return stats dict: `{"detail_count": N, "summary_count": M, "categories": [...]}`
9. Log: "Ingested trader {address[:8]}...: {N} detail trades, {M} summary categories"

`run_full_sweep() -> dict`:
1. Call `ingest_active_markets()`
2. For each active market with a detail category (use CategoryFilter), call `discover_traders_from_market()`
3. For each newly discovered trader (backfill_complete=False), call `ingest_trader_history()`
4. Return overall stats: markets ingested, traders discovered, trades stored
5. Log summary at end

**Error handling:**
- Wrap each trader's ingestion in try/except to not fail the entire sweep on one trader
- Log errors and continue
- Use database transactions per-trader (commit after each trader, not after entire sweep)

**Dedup strategy:**
- Markets: upsert on condition_id (use `session.merge()` or check-then-insert)
- Trades: check trade_id uniqueness, skip duplicates
- Summaries: upsert on (trader_address, category) unique constraint

Create `src/pipeline/queries.py` with query functions using SQLAlchemy 2.0 `select()` syntax:

`get_trades_by_date_range(session, start_date: datetime, end_date: datetime, trader_address: str | None = None) -> list[Trade]`:
- Query `trades` table with `timestamp BETWEEN start_date AND end_date`
- Optionally filter by trader_address
- Order by timestamp DESC
- Uses composite index `ix_trade_trader_timestamp`
- Returns list of Trade ORM objects

`get_trades_by_resolution_status(session, resolved: bool, trader_address: str | None = None) -> list[Trade]`:
- Join `trades` with `markets` on market_id == condition_id
- Filter where market.outcome IS NOT NULL (resolved=True) or IS NULL (resolved=False)
- Optionally filter by trader_address
- Returns list of Trade ORM objects

`get_trader_trades(session, trader_address: str, category: str | None = None) -> list[Trade]`:
- Query trades for a specific trader
- Optionally filter by category (join with markets table)
- Order by timestamp DESC
- Returns list of Trade ORM objects

`get_trader_summary(session, trader_address: str) -> list[TraderCategorySummary]`:
- Query all category summaries for a trader
- Returns list of TraderCategorySummary ORM objects

`get_active_markets(session, category: str | None = None) -> list[Market]`:
- Query markets where active=True
- Optionally filter by category
- Returns list of Market ORM objects
