Implement one public function:

```python
def upsert_gamma_events(events: list[dict], session: Session) -> int:
    """Upsert Gamma API event dicts into the gamma_events table.

    Args:
        events: Raw event dicts from GammaMarketClient.get_closed_esports_events()
        session: Active SQLAlchemy session (caller manages commit)

    Returns:
        Number of rows upserted (inserted or updated)
    """
```

Implementation details:
1. For each event dict, extract the following fields:
   - `event_id = str(event.get("id", ""))` — skip if empty
   - `title = str(event.get("title", "") or "")[:500]`
   - `slug = str(event.get("slug", "") or "")[:200]`
   - `outcome_prices = json.dumps(event.get("outcomePrices") or [])` — outcomePrices is a list like `["0.99", "0.01"]` at event level
   - `tags = json.dumps(event.get("tags") or [])` — list of tag dicts, each has `id`, `slug`, `label`
   - `clob_token_ids`: Collect from nested `markets[]` — for each market in `event.get("markets") or []`, parse `market.get("clobTokenIds")` (may be a JSON string or list), accumulate into a flat list of unique token ID strings. Store as `json.dumps(sorted_unique_list)`.
   - `start_date`: Parse `event.get("startDate")` — use `datetime.fromisoformat(s.replace("Z", "+00:00"))` if not None, else None. Store as ISO string for SQLite.
   - `end_date`: Same pattern as start_date.

2. Use a raw SQL upsert for efficiency (same pattern as TokenCatalogBuilder):
   ```python
   session.execute(
       text("""
           INSERT OR REPLACE INTO gamma_events
             (event_id, title, slug, outcome_prices, clob_token_ids, tags,
              start_date, end_date, created_at, updated_at)
           VALUES
             (:event_id, :title, :slug, :outcome_prices, :clob_token_ids, :tags,
              :start_date, :end_date, :now, :now)
       """),
       rows,
   )
   ```
   Where `rows` is a list of dicts and `:now` is `datetime.utcnow().isoformat()`.

3. Skip events with empty `event_id` and log a warning.
4. Log: `logger.info(f"Upserted {len(rows)} gamma events into gamma_events table")`
5. Return `len(rows)`.

Use `INSERT OR REPLACE` (not `INSERT OR IGNORE`) so that re-runs update existing rows with fresh data (idempotent update semantics). The `event_id` primary key ensures uniqueness.

Do NOT call `session.commit()` inside this function — the caller (CLI command) manages the transaction boundary.
