## Implementation Plan

### 1. Discovery Tasks

```bash
# Audit CLOB API /tokens endpoint
curl -s "https://clob.polymarket.com/api/tokens" | jq '. | length'

# Check current token catalog size
psql polymarket -c "SELECT COUNT(*) FROM token_catalog;"

# Check graph_ trade distribution
psql polymarket -c "
  SELECT COUNT(*), 
         SUBSTRING(market_id FROM 1 FOR 20) as market_prefix
  FROM trades 
  WHERE market_id LIKE 'graph_%'
  GROUP BY market_prefix
  ORDER BY COUNT DESC
  LIMIT 10;
"
```

### 2. Implementation Tasks

#### 2.1 Add Token API Methods (`src/api/client.py`)

Add methods to fetch tokens from CLOB API:
- `get_tokens(limit=None)` — Fetch all tokens or paginated batch
- `get_token_by_id(token_id)` — Fetch single token metadata

Response format expected:
```json
{
  "condition_id": "0x...",
  "question": "Will X happen?",
  "tokens": [
    {"token_id": "5410...", "outcome": "YES"},
    {"token_id": "5411...", "outcome": "NO"}
  ]
}
```

#### 2.2 Build CLI Command (`src/cli/commands.py`)

Add `build-token-catalog` command:
```python
@click.command()
@click.option("--batch-size", default=100, help="Tokens per batch")
@click.option("--verbose", is_flag=True, help="Verbose logging")
def build_token_catalog(batch_size, verbose):
    """Build complete token catalog from CLOB API."""
```

Features:
- Fetches all tokens from CLOB API
- Upserts into `token_catalog` table (idempotent)
- Progress logging for large builds
- Handles rate limiting

#### 2.3 Update TokenCatalog Model (if needed)

Check if `src/db/models.py` TokenCatalog model needs new fields for CLOB API data.
