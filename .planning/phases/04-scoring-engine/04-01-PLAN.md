Purpose: Concentration is a core component (~25% weight) of the expertise score. It measures how focused a trader is within eSports overall (eSports-level) and within a specific game (game-level). This enables the "niche hypothesis" — traders deeply focused in one game likely have domain knowledge generalists don't.

Output: Pure-function concentration module with SpecializationProfile dataclass, tested via TDD.

<feature>
  <name>Two-tier concentration metrics and specialization classification</name>
  <files>src/evaluation/concentration.py, tests/test_concentration.py</files>
  <behavior>
    Module follows established pure-function, duck-typed pattern from metrics.py and consistency.py.
    All calculations use Decimal arithmetic. No SQLAlchemy imports.

    **SpecializationProfile frozen dataclass:**
    - esports_level: str ("specialist" or "generalist")
    - game_level: str ("specialist" or "generalist")
    - esports_concentration: Decimal (0-1, fraction of total volume in eSports)
    - game_concentration: Decimal (0-1, fraction of eSports volume in this game)
    - primary_game: str | None (game slug with highest concentration, None if generalist)

    **Functions:**

    1. `calculate_esports_concentration(esports_volume: Decimal, total_volume: Decimal) -> Decimal`
       - Returns fraction of total volume in eSports (0-1)
       - Returns Decimal("0") if total_volume is 0
       - Input: raw Decimal volumes (pre-computed by caller using calculate_total_volume)

    2. `calculate_game_concentration(game_volume: Decimal, esports_volume: Decimal) -> Decimal`
       - Returns fraction of eSports volume in specific game (0-1)
       - Returns Decimal("0") if esports_volume is 0
       - Input: raw Decimal volumes

    3. `classify_specialization(esports_concentration, game_concentration, game_slug, esports_threshold=Decimal("0.7"), game_threshold=Decimal("0.5")) -> SpecializationProfile`
       - Classifies at both levels using configurable thresholds
       - esports_threshold default 0.7 (70% of total volume in eSports = eSports specialist)
       - game_threshold default 0.5 (50% of eSports volume in one game = game specialist)
         NOTE: Game threshold lower than eSports threshold because a trader active in 2 games can be specialist in both at 50%+ each
       - primary_game set to game_slug if game_level == "specialist", else None

    **Test cases (RED phase):**
    - Pure eSports specialist: esports_concentration=0.9, game_concentration=0.8 -> specialist/specialist
    - eSports specialist, game generalist: esports_concentration=0.85, game_concentration=0.3 -> specialist/generalist
    - eSports generalist: esports_concentration=0.3 -> generalist at eSports level regardless of game
    - Zero total volume: returns Decimal("0") concentration
    - Zero eSports volume: game concentration = Decimal("0")
    - Boundary: exactly at threshold (0.7 eSports, 0.5 game) -> specialist (>= comparison)
    - Multi-game specialist: same trader can be specialist in CS2 AND Valorant (independent per call)
    - primary_game set correctly: specialist has game_slug, generalist has None
  </behavior>
