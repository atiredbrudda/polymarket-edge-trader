  <behavior>
  **PositionData dataclass (src/discovery/position_tracker.py):**
  - market_id: str
  - trader_address: str
  - size: Decimal (net shares - positive=long, negative=short, zero=flat)
  - direction: str ("LONG", "SHORT", or "FLAT")
  - avg_entry_price: Decimal | None (weighted average: sum(buy_size * buy_price) / sum(buy_size) for longs)
  - entry_timestamp: datetime | None (timestamp of first trade opening current position)
  - total_cost_basis: Decimal (sum of size*price across opening trades)
  - trade_count: int (number of trades in history)
  - first_trade_timestamp: datetime | None (earliest trade, for entry timing relative to market creation)
  - last_trade_timestamp: datetime | None (latest trade)

  **calculate_position(trades: list) -> PositionData:**
  Pure function. Accepts list of trade-like objects (anything with side, size, price, timestamp, market_id, trader_address attributes - works with both ORM Trade objects and test stubs).

  Algorithm:
  1. Sort trades by timestamp ascending (chronological order)
  2. Initialize net_size=0, total_cost_basis=0, entry_timestamp=None
  3. For each trade:
     - BUY: net_size += size, total_cost_basis += size * price
       - If this is the first BUY after a flat position, record entry_timestamp
     - SELL: net_size -= size, total_cost_basis -= size * price
       - If net_size reaches 0 (full closure), reset entry_timestamp=None and total_cost_basis=0
       - If position flips (was long, now short), handle the flip correctly
  4. Compute direction from net_size sign
  5. Compute avg_entry_price = total_cost_basis / abs(net_size) if net_size != 0, else None
  6. Return PositionData with all computed fields

  CRITICAL: All arithmetic uses Decimal. Initialize Decimal from strings: Decimal("0"), not Decimal(0.0).

  Edge cases to handle:
  - Empty trade list: raise ValueError("No trades provided")
  - All trades same side (only buys): valid long position
  - Full closure then reopen: entry_timestamp resets to new opening trade
  - Partial closure: position reduced, cost basis reduced proportionally
  - Multiple fills at different prices: weighted average is correct
  - Single trade: position equals that trade's size/price

  **calculate_pnl(position: PositionData, resolution_price: Decimal, market_outcome: str) -> dict:**
  Pure function. Computes PnL for a resolved position.

  Args:
    - position: PositionData from calculate_position
    - resolution_price: Final settlement price (Decimal, typically 0 or 1 for binary markets)
    - market_outcome: "YES", "NO", or "VOID"

  Returns dict:
    - outcome: "win", "loss", "void", or "flat"
    - pnl: Decimal (profit/loss amount)
    - return_pct: Decimal | None (pnl / total_cost_basis as percentage, None if flat/void)

  Logic:
    - VOID: pnl=0, outcome="void"
    - FLAT position: pnl=0, outcome="flat"
    - LONG: pnl = size * (resolution_price - avg_entry_price)
    - SHORT: pnl = abs(size) * (avg_entry_price - resolution_price)
    - outcome = "win" if pnl > 0, "loss" if pnl < 0, "flat" if pnl == 0

  <implementation>
  1. Create src/discovery/__init__.py (empty or with key exports)
  2. Implement PositionData as a frozen dataclass (immutable - pure function output)
  3. Implement calculate_position as a standalone function (not a class method)
  4. Implement calculate_pnl as a standalone function
  5. Use a simple protocol or duck-typing for trade input (don't import ORM Trade directly - accept anything with the right attributes). This keeps the module decoupled from SQLAlchemy.
  6. Tests use simple dataclass stubs for trade objects, not real ORM models

  **Design choice:** Functions, not classes. Position tracking is a pure computation with no state. calculate_position(trades) and calculate_pnl(position, price, outcome) are standalone functions that take data in and return data out. No PositionTracker class needed.

  **Decimal discipline:**
  - All Decimal literals from strings: Decimal("0"), Decimal("1"), Decimal("0.000001")
  - Use quantize for display: price.quantize(Decimal("0.000001"))
  - Never mix float and Decimal in same expression

  **Anti-patterns to avoid:**
  - No incremental state - ALWAYS recompute from full trade list
  - No float arithmetic - ALL Decimal
  - No SQLAlchemy imports - keep this module pure (duck-typing for trade input)
  - No mutation of input trades list
