<objective>
Build the YAML-based taxonomy system for eSports market classification. This implements TAXO-01 (YAML definitions), TAXO-02 (keyword matching classification), and TAXO-03 (YAML-only extensibility).

Purpose: Enable the system to classify Polymarket markets into a 4-level eSports hierarchy using regex pattern matching against market titles, with the deepest matching node winning. Market type (match vs prop) is detected as a tag attribute. Unknown entities are flagged for review.

Output: Taxonomy Pydantic models, YAML loader, pattern-matching classifier, market type detector, seed eSports YAML, and comprehensive tests.
</objective>

  <behavior>
  **Taxonomy Models (src/taxonomy/models.py):**
  - TeamNode: name (str), patterns (List[str] - regex patterns), aliases (Optional[List[str]])
  - TournamentNode: name (str), patterns (List[str]), tier (Optional[str] - e.g. "major", "minor"), teams (List[TeamNode])
  - GameNode: name (str), patterns (List[str]), tournaments (List[TournamentNode])
  - TaxonomyConfig: name (str, default "eSports"), games (List[GameNode])
  - All models are Pydantic BaseModel with proper validation

  **Taxonomy Loader (src/taxonomy/loader.py):**
  - load_taxonomy(path: Path) -> TaxonomyConfig
  - Uses yaml.safe_load() (NEVER yaml.load()) for security
  - Validates parsed YAML against TaxonomyConfig Pydantic schema
  - Raises clear ValidationError on malformed YAML with field paths
  - Raises FileNotFoundError on missing file
  - Raises yaml.YAMLError on invalid YAML syntax

  **Classifier (src/taxonomy/classifier.py):**

  ClassificationResult dataclass:
  - node_path: str (e.g. "eSports.CS2.IEM Katowice.NaVi")
  - depth: int (0=root, 1=game, 2=tournament, 3=team)
  - game: str | None
  - tournament: str | None
  - team: str | None
  - market_type: str | None ("match" or "prop")
  - matched_pattern: str (the pattern that matched)
  - flagged_for_review: bool

  PatternMatcher class:
  - __init__(taxonomy: TaxonomyConfig): Precompiles ALL regex patterns from taxonomy into flat list with depth/path metadata. re.IGNORECASE on all patterns.
  - classify(market_title: str) -> ClassificationResult | None: Tests all precompiled patterns, returns deepest match. If multiple patterns match at same depth, first match in taxonomy order wins. Returns None for completely unmatched titles.
  - classify_with_review(market_title: str) -> ClassificationResult: Always returns a result. If no taxonomy match, returns flagged_for_review=True. If partial match (e.g. game matched but title contains "vs" suggesting team match), also flags for review.

  detect_market_type(market_title: str) -> str | None:
  - Precompiled patterns for "match" type: \bvs\.?\b, \bv\b, \b-\b (with team name context), \b@\b
  - Precompiled patterns for "prop" type: \bwinner\b, \btop\s+\d+\b, \bover\s+\d+\.?\d*\b, \bchampion\b, \bMVP\b
  - Returns "match" if match patterns score higher, "prop" if prop patterns score higher, None if ambiguous
  - Standalone function, not class method

  **Seed Taxonomy (data/taxonomy/esports.yaml):**
  4 games minimum: CS2, Dota 2, League of Legends, Valorant
  Each game has 2-3 tournament patterns and 3-5 team patterns (per user's discretion - teams duplicated per game for simpler path queries)
  Example teams across games: NaVi (CS2, Dota2), Team Liquid (CS2, Dota2, LoL), FaZe Clan (CS2, Valorant), T1 (LoL, Valorant), Cloud9 (CS2, LoL, Valorant)
