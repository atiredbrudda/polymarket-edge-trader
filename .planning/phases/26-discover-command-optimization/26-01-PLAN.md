# Plan 26-01: Research & Profiling

## Goal

Profile the current `polymarket discover` command to understand exact timing breakdown and validate hypotheses about performance bottlenecks.

## Tasks

### 1. Profile Command Execution

Run instrumented discovery to measure:
- Time spent in `ingest_targeted_markets()` (Gamma API calls)
- Time spent in `discover_traders_from_market()` (Polymarket API calls)
- Time spent in `matcher.match()` (regex classification)
- Time spent in `extract_entities()` (LLM calls + retries)
- Number of regex matches vs LLM fallbacks

**Commands:**
```bash
# Add timing instrumentation
time polymarket discover --niche esports --closing-within 1h 2>&1 | tee /tmp/discover_profile.log

# Check database after run
sqlite3 data/polymarket.db "SELECT COUNT(*) FROM market_entities WHERE extracted_at > datetime('now', '-5 minutes');"
```

### 2. Measure Regex Match Rate

Query existing data to determine regex effectiveness:

```sql
-- Total esports markets with entity extraction
SELECT COUNT(*) FROM market_entities 
WHERE game IS NOT NULL AND game != '';

-- Sample questions to analyze pattern coverage
SELECT question, team_a, team_b, game, tournament 
FROM market_entities 
JOIN markets ON market_entities.condition_id = markets.condition_id
WHERE markets.category LIKE '%esports%'
ORDER BY market_entities.extracted_at DESC
LIMIT 20;
```

### 4. Analyze LLM Error Patterns

Check logs for 529 error frequency:

```bash
# Search for LLM errors in recent runs
grep -i "overloaded\|529\|extraction.*failed" logs/*.log | tail -50

# Count error types
grep -o "529\|429\|503" logs/*.log | sort | uniq -c
```
