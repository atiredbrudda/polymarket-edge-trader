  <action>
Create `src/signals/pipeline.py` following the established orchestration pattern from `src/pipeline/scoring_pipeline.py`.

**Dataclass: `SignalResult`** (frozen=True)
- market_id: str
- direction: str — "LONG" or "SHORT"
- confidence_score: Decimal — 0-100
- expert_count: int
- total_experts_in_market: int
- agreement_percentage: Decimal
- expert_addresses: list[str]
- first_mover_address: str | None
- follower_classifications: dict[str, str] — {address: "first_mover"|"fast_follower"|"independent"}
- herding_status: str — always "not_analyzed" in Phase 5 (stub)
- status: str — "active" or "inactive"
- computed_at: datetime

**Function 1: `refresh_market_signal(session, market_id, min_experts=3, min_agreement_pct=Decimal("75"), now=None)`**
Pipeline flow:
1. Query expert positions for the market using `get_expert_positions_for_market(session, market_id)`
2. Query latest expertise scores for all traders in those positions. Use max(computed_at) subquery to get latest score per trader. Build dict {trader_address: raw_score}.
3. Call `detect_consensus(positions, expert_scores, min_experts, min_agreement_pct)` from detection.py
4. For each ConsensusResult:
   a. Call `calculate_confidence_score(result.expert_positions, result.total_experts_in_market, min_experts)` from confidence.py
   b. Call `classify_followers(result.expert_positions, result.first_mover_address)` from detection.py
   c. Determine status: "active" if confidence > 0, "inactive" otherwise
   d. Create SignalSnapshot ORM object and session.add() (append-only INSERT)
   e. Build SignalResult dataclass
5. Handle "signal lost" case: If a market previously had an active signal for a direction but no longer meets consensus thresholds, INSERT a new SignalSnapshot with confidence_score=0, status="inactive", expert_count=0. This marks the signal as lost (the previous active snapshot remains in history for Phase 6 delta detection).
6. session.commit()
7. Return list[SignalResult]

For step 5 "signal lost" detection: Query get_signal_history(session, market_id) to find previously active directions. If a direction was active in the last snapshot but doesn't appear in current consensus results, create an inactive snapshot for it.

**Function 2: `refresh_all_signals(session, window_hours=24, min_experts=3, now=None)`**
1. Call `get_markets_by_expert_activity(session, window_hours, min_experts=1)` to find markets with expert activity
2. For each market, call `refresh_market_signal(session, market_id, min_experts, now=now)`
3. Collect all SignalResults
4. Return list[SignalResult] sorted by confidence_score DESC

**Function 3: `get_ranked_signals(session, window_hours=None, min_confidence=None, limit=50)`**
1. Call `get_latest_signals(session, status="active", min_confidence=min_confidence, limit=limit)` to get latest active signals
2. If window_hours is provided, filter to signals where the market has expert activity within that window (use get_markets_by_expert_activity as a filter)
3. Return list sorted by confidence_score DESC
4. This provides the 1h/6h/24h time-window views per SGNL-04

**Function 4: `assess_herding(signal_result)` — STUB**
- Returns "not_analyzed" always
- Docstring explains: "Herding detection deferred per user decision. All expert positions count equally regardless of timing proximity. See Phase 5 CONTEXT.md."
- This satisfies SGNL-03 formally with a minimal stub
