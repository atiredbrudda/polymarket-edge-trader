  <name>Task 2: Confidence score calculation (TDD)</name>

Create `src/signals/confidence.py` with pure function for 0-100 confidence scoring. Follows project Decimal precision patterns.

**Function: `calculate_confidence_score(experts_agreeing, experts_total, min_experts=3)`**
- Input: list of position-like objects (experts in same direction), total expert count in market (int), min_experts (int)
- Returns Decimal (0-100)
- Returns Decimal("0") if len(experts_agreeing) < min_experts

**Formula (from research, user-decided):**
- **Agreement component (60% weight):** agreement_pct = len(experts_agreeing) / experts_total * 100. This ranges 0-100. Weight: 0.6
- **Sample size component (30% weight):** Uses asymptotic formula: `(1 - exp(-(n - min_experts) / 10)) * 100` where n = len(experts_agreeing). At exactly min_experts this is 0; at min+10 experts it's ~63; asymptotes to 100. Weight: 0.3. Use manual exp approximation or math.exp converted to Decimal (since this is not financial arithmetic, math.exp -> Decimal conversion is acceptable here).
- **Position-size uniformity component (10% weight, 0-10 points):** Calculate coefficient of variation (CV) of position volumes across agreeing experts. Volume = abs(size * avg_entry_price) if avg_entry_price else abs(size). Uniformity = (1 - min(CV, 1)) * 10. Uniform positions (CV near 0) give up to 10 points. Single expert or zero volume gives 0 points.
- **Final:** confidence = agreement * 0.6 + sample_size * 0.3 + uniformity. Cap at 100.

**Helper: `_compute_position_volume(position)` (matches scoring_pipeline.py pattern exactly)**
