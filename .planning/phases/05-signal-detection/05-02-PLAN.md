Create `src/signals/queries.py` with signal-specific database query functions. If `src/signals/__init__.py` doesn't exist yet (Plan 01 may not have run), create it.

**Function 1: `get_latest_signals(session, status="active", min_confidence=None, limit=50)`**
- Returns latest SignalSnapshot per (market_id, direction) using max(computed_at) subquery
- Filter by status ("active", "inactive", or None for all)
- Optional min_confidence filter
- Ordered by confidence_score DESC
- Limit results
- Returns list of SignalSnapshot ORM objects

**Function 2: `get_signal_history(session, market_id, direction=None, limit=20)`**
- Returns all SignalSnapshot rows for a market, ordered by computed_at DESC
- Optional direction filter
- Used for tracking strength changes over time (Phase 6 alerting)

**Function 3: `get_expert_positions_for_market(session, market_id, min_score=Decimal("70"))`**
- Returns list of Position ORM objects for experts (score >70) in a specific market
- Joins Position -> ExpertiseScore (latest via max computed_at subquery)
- Filters: direction IN ("LONG", "SHORT"), ExpertiseScore.raw_score > min_score
- This is the database-backed version of the pure function's input assembly

**Function 4: `get_markets_by_expert_activity(session, window_hours=24, min_experts=1)`**
- Returns markets that have expert (score >70) position activity within the time window
- Joins Position -> ExpertiseScore (latest scores)
- Filters: Position.last_trade_timestamp >= now - window_hours
- Groups by market_id, counts distinct expert traders
- Filters: expert_count >= min_experts
- Returns list of tuples: (market_id, expert_count, latest_activity_timestamp)
- Ordered by expert_count DESC, latest_activity DESC

Follow existing query patterns from `src/pipeline/queries.py`:
- Use `select()` with explicit joins
- Use `func.max()` subqueries for latest scores (matching get_game_leaderboard pattern)
- Use `datetime.now(UTC)` for time calculations (not datetime.utcnow())
- Return typed lists
