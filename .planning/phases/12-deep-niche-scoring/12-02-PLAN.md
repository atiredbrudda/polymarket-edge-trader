**Implementation approach — reuse compute_game_scores pattern but with depth-aware changes:**

1. **Position query**: Use `get_positions_for_slug(session, slug)` instead of `get_positions_for_game`. This returns positions for the slug and all its children (same LIKE pattern).

2. **Group by trader**: Same as compute_game_scores.

3. **Concentration calculation per trader** — varies by depth:
   - depth=1 (game): Same as existing — `calculate_game_concentration(game_volume, esports_volume)`
   - depth=2 (tournament): Get game volume from positions under parent game slug (extract parent: `slug.rsplit('.', 1)[0]`), then `calculate_tournament_concentration(tournament_volume, game_volume)`
   - depth=3 (team): Get tournament volume from positions under parent tournament slug, then `calculate_team_concentration(team_volume, tournament_volume)`

   For the `game_concentration` parameter passed to `calculate_expertise_score`, use the depth-appropriate concentration. The scoring function is depth-agnostic — it just uses whatever concentration is passed.

   For `esports_concentration`, always use the same trader-wide calculation (esports_volume / total_volume). This is a trader-level metric, not depth-specific.

4. **Score computation**: Call `calculate_expertise_score()` with:
   - positions: filtered to this slug
   - game_slug: the full slug (e.g., "esports.cs2.iem-katowice")
   - game_concentration: the depth-appropriate concentration value
   - All other params same as compute_game_scores

5. **Percentile normalization**: Normalize within the same slug+depth population (not mixed with other depths).

6. **Persistence**: Create ExpertiseScore rows with `taxonomy_depth=taxonomy_depth` set correctly.

7. **Leaderboard build**: Same as compute_game_scores.

Also add `compute_all_taxonomy_scores`:
```python
def compute_all_taxonomy_scores(
    session: Session,
    depth: int,
    weights: dict[str, Decimal] | None = None,
    now: datetime | None = None,
) -> dict[str, list[LeaderboardEntry]]:
    """Compute scores for all slugs at a given taxonomy depth."""
    slugs = get_all_slugs_with_positions_at_depth(session, depth)
    results = {}
    for slug in slugs:
        leaderboard = compute_taxonomy_scores(session, slug, depth, weights=weights, now=now)
        results[slug] = leaderboard
    return results
```

**Function signature:**
```python
def identify_hidden_specialists(
    session: Session,
    game_slug: str,
    game_score_threshold: Decimal = Decimal("60"),
    deep_score_threshold: Decimal = Decimal("75"),
) -> list[dict]:
```

**Algorithm:**
1. Query all ExpertiseScore rows for `game_slug` at depth=1 where raw_score < game_score_threshold. Use max(computed_at) subquery to get latest scores only.
2. For each such trader, query their ExpertiseScore rows at depth=2 and depth=3 where the slug starts with `game_slug` (i.e., tournament/team scores under that game). Use max(computed_at) subquery for latest.
3. Filter to traders who have at least one deep score >= deep_score_threshold.
4. Return list of dicts:
```python
{
    "trader_address": str,
    "game_slug": str,
    "game_score": Decimal,
    "deep_slug": str,  # e.g., "esports.cs2.iem-katowice"
    "deep_depth": int,  # 2 or 3
    "deep_score": Decimal,
    "score_delta": Decimal,  # deep_score - game_score
}
```
5. Sort by score_delta DESC (biggest specialists first).
6. A trader may appear multiple times if they specialize at multiple deep slugs — that's expected.

**Tests** in `tests/test_scoring_pipeline_deep.py`:
1. Test hidden specialist found: trader with game_score=50, tournament_score=80 → returned

3. Test deep generalist excluded: trader with game_score=40, tournament_score=40 → not returned
4. Test multiple specialists: trader specializing in 2 different tournaments → 2 entries
5. Test empty result when no specialists exist
6. Test sort order: highest score_delta first
