   ```python
   def format_research_table(trades_data: list[dict], trader_address: str, total_count: int) -> Table:
       """Format JBecker trade history as Rich table.

       Args:
           trades_data: List of trade dicts from JBeckerDataset (raw DuckDB output)
           trader_address: Trader address being queried
           total_count: Total trades found (may differ from len(trades_data) if truncated)

       Returns:
           Rich Table renderable
       """
   ```

   Table columns:
   - "#" (row number, right-justified, dim style)
   - "Timestamp" (human-readable from Unix timestamp)
   - "Role" (MAKER or TAKER based on trader address match)
   - "Side" (BUY/SELL with green/red color)
   - "Size (USDC)" (makerAmountFilled or takerAmountFilled / 1e6, based on role)
   - "Price" (from price field)
   - "Block" (blockNumber, dim style)

   Table title: `f"Trade History: {truncate_address(trader_address)} ({total_count} trades)"`

   If `len(trades_data) < total_count`, add footer: `f"Showing {len(trades_data)} of {total_count} trades"`

   Role determination: if `trade["maker"].lower() == trader_address.lower()` -> "MAKER" else "TAKER"
   Size: if MAKER, use makerAmountFilled/1e6; if TAKER, use takerAmountFilled/1e6

1. **Add `research` command to `src/cli/commands.py`:**

   ```python
   @cli.command()
   @click.argument("address")
   @click.option("--format", "-f", "output_format", type=click.Choice(["table", "json", "csv"]), default="table",
                 help="Output format")
   @click.option("--limit", "-l", default=50, type=int, help="Max trades to display (default 50)")
   @click.option("--verbose", "-v", is_flag=True, help="Enable debug logging")
   def research(address, output_format, limit, verbose):
       """Query full trade history from JBecker dataset.

       Queries the complete historical dataset (2020-2026) offline without API rate limits.
       Requires JBecker dataset downloaded and JBECKER_DATA_PATH configured.

       ADDRESS can be full address or prefix (e.g., 0xeffd76).

       \b
       Examples:
           polymarket research 0xeffd76
           polymarket research 0xeffd76 --format json
           polymarket research 0xeffd76 --format csv --limit 1000
       """
   ```

   Implementation:
   a. Setup logging (verbose flag)
   b. Get settings via `get_settings()`
   c. Create `JBeckerDataset(settings.jbecker_data_path)` -- import from src.datasources.jbecker
   d. Check `jbecker.is_available()`:
      - If False: print user-friendly download instructions using Rich markup:
        ```
        [red]JBecker dataset not available.[/red]

        [yellow]To download and setup:[/yellow]
        1. wget https://s3.jbecker.dev/data.tar.zst  (33.5 GB)
        2. tar --use-compress-program=zstd -xvf data.tar.zst
        3. Set JBECKER_DATA_PATH in .env to point to the data/ directory
        4. Verify: ls $JBECKER_DATA_PATH/polymarket/trades/
        ```
        Return early.
   e. Resolve partial address using `find_trader_by_prefix(session, address)` if DB available,
      otherwise use address directly (JBecker queries work with any address, DB lookup is optional)
   f. Get total count: `jbecker.get_trade_count(full_address)`
   g. Query trades with limit: `jbecker.query_trader_history(full_address, limit=limit)`
   h. Format output based on --format:
      - "table": use `format_research_table(trades, full_address, total_count)` + console.print
      - "json": `json.dumps(trades, indent=2, default=str)` to console
      - "csv": Use csv.DictWriter to stdout with keys from first trade
   i. Log command completion

   ```python
   @cli.command("batch-analyze")
   @click.option("--addresses", "-a", multiple=True, help="Trader addresses to analyze")
   @click.option("--file", "-f", "address_file", type=click.Path(exists=True), help="File with addresses (one per line)")
   @click.option("--verbose", "-v", is_flag=True, help="Enable debug logging")
   def batch_analyze(addresses, address_file, verbose):
       """Bulk ingest trader histories from JBecker dataset.

       Ingests complete trade histories for multiple traders at once,
       storing them in the database for analysis. Uses batch deduplication.

       Provide addresses via --addresses flag (repeatable) or --file flag.

       \b
       Examples:
           polymarket batch-analyze -a 0xeffd76... -a 0xeefa8e...
           polymarket batch-analyze --file traders.txt
       """
   ```

   Implementation:
   a. Collect addresses from both sources (--addresses and --file)
      - If --file: read lines, strip whitespace, skip empty/comment lines (starting with #)

      - If no addresses: print error and return
   b. Get dependencies via `_get_dependencies()`
   c. Create `JBeckerDataset(settings.jbecker_data_path)`

      - On error: log warning, add error entry to results, continue to next trader
   g. After all traders: display `format_batch_summary(results)`
   h. Print totals: total trades ingested, total duplicates skipped, total errors
