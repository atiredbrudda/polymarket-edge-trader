**GREEN phase - Implement `src/datasources/converters.py`:**

Create `jbecker_trade_to_api_response(jbecker_trade: dict, trader_address: str) -> TradeResponse`:

Follow the EXACT pattern from `src/graph/converters.py::graph_trade_to_api_response()`:

1. Normalize addresses: `trader_address.lower()`, `maker.lower()`, `taker.lower()`
2. Determine role: `is_maker = (trader_address == maker)`
3. Convert amounts from 6-decimal integers:
   - `maker_amount = Decimal(str(jbecker_trade["makerAmountFilled"])) / Decimal("1000000")`
   - `taker_amount = Decimal(str(jbecker_trade["takerAmountFilled"])) / Decimal("1000000")`
   - IMPORTANT: Wrap in str() first to handle both string and numeric types from DuckDB
4. Determine side and size based on role:
   - Maker: side=original, size=maker_amount, asset_id=makerAssetId
   - Taker: side=OPPOSITE, size=taker_amount, asset_id=takerAssetId
5. Parse price: `Decimal(str(jbecker_trade["price"]))` -- do NOT catch ValueError, let TradeResponse validator handle invalid prices
6. Convert timestamp: `datetime.fromtimestamp(int(jbecker_trade["timestamp"]))`
7. Determine market_id: `f"jbecker_{jbecker_trade['transactionHash']}_{asset_id}"`
8. Determine asset_ticker: odd asset_id int -> "YES", even -> "NO", fallback "UNKNOWN"
9. Return `TradeResponse(id=..., market=market_id, trader=trader_address, side=side, size=size, price=price, timestamp=timestamp, asset_ticker=asset_ticker)`
