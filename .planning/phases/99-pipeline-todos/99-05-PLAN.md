    ADD this helper near the top of backfill_trader(), before the trade processing loop:
    ```python
    def _normalize_ts(ts) -> str:
        """Return ISO timestamp truncated to seconds. Prevents API/Graph precision mismatch."""
        if isinstance(ts, int):
            return datetime.fromtimestamp(ts, tz=timezone.utc).replace(microsecond=0).isoformat()
        if ts:
            try:
                dt = datetime.fromisoformat(str(ts).replace("Z", "+00:00"))
                return dt.replace(microsecond=0).isoformat()
            except Exception:
                pass
        return datetime.now(timezone.utc).replace(microsecond=0).isoformat()
    ```

    REPLACE the existing timestamp_iso assignment block (~lines 371-381):
    ```python
    # BEFORE:
    if isinstance(timestamp, int):
        try:
            timestamp_iso = datetime.fromtimestamp(timestamp, tz=timezone.utc).isoformat()
        except (ValueError, OSError):
            timestamp_iso = now
    else:
        timestamp_iso = timestamp if timestamp else now

    # AFTER:
    timestamp_iso = _normalize_ts(timestamp)
    ```

    ```python
    # BEFORE (top of trade processing, ~line 310):

    # AFTER:
    now = datetime.now(timezone.utc).replace(microsecond=0).isoformat()
    ```

    Rationale: Data API returns ISO strings that may include microseconds; Graph returns

    ```python

    if trade_batch:
        try:
            db["trades"].insert_all(trade_batch, ignore=True)
            stats["ingested"] += len(trade_batch)
        except Exception:
            # Batch failed — fall back to individual inserts

                try:
                    db["trades"].insert(item, replace=False)
                    stats["ingested"] += 1
                except Exception:
                    stats["skipped"] += 1
    ```

    NOTE: sqlite-utils insert_all with ignore=True silently skips duplicate trade_ids
    (equivalent to INSERT OR IGNORE). The fallback ensures correctness if batch fails.

    Rationale: 4,513 individual inserts → ~46 batch inserts. Reduces DB write overhead by ~80%.
