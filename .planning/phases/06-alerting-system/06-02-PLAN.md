<feature>
  <name>Alert Formatter</name>
  <files>src/alerts/formatter.py, tests/test_alert_formatter.py</files>
  <behavior>
    The formatter transforms signal data into rich Telegram HTML messages.

    Primary function signature:
    ```python
    def format_signal_alert(
        event_type: str,
        market_question: str,
        signal: SignalResult,
        expert_positions: list[dict] | None = None,
    ) -> str
    ```

    Where expert_positions is a list of dicts with keys: address, size, direction, avg_entry_price.
    This data comes from querying Position table for signal.expert_addresses (done by caller).

    Output format (Telegram HTML parse_mode):
    - Event type header with icon: NEW, STRENGTHENING, WEAKENING, LOST
    - Market question (HTML-escaped)
    - Direction and confidence score
    - Expert agreement metrics (count/total, percentage)
    - First-mover address (truncated, monospace)
    - Fast-follower count (from follower_classifications)
    - Expert addresses (first 5 truncated, with "+N more" if > 5)
    - Individual position sizes (from expert_positions parameter)
    - Polymarket link

    Cases (input -> expected output characteristics):
    1. NEW event -> header contains "NEW CONSENSUS" with icon
    2. STRENGTHENING event -> header contains "STRENGTHENING" with upward icon
    3. WEAKENING event -> header contains "WEAKENING" with downward icon
    4. LOST event -> header contains "LOST CONSENSUS" with icon
    5. Market question with HTML chars (<, >, &) -> properly escaped in output
    6. 3 expert addresses -> all shown, no "+N more"
    7. 7 expert addresses -> first 5 shown, "+2 more" indicator
    8. No first_mover_address (None) -> first mover section omitted
    9. No expert_positions (None) -> position sizes section omitted
    10. Expert positions provided -> each address shown with size and direction
    11. Output contains valid Telegram HTML tags: b, i, code, a href, pre

    Helper function for position data retrieval:
    ```python
    def get_expert_position_details(
        session: Session,
        market_id: str,
        expert_addresses: list[str],
    ) -> list[dict]
    ```

    Queries Position table for expert addresses in a given market.
    Returns list of dicts: {"address": str, "size": Decimal, "direction": str, "avg_entry_price": Decimal | None}
  </behavior>
  <implementation>
    1. Create src/alerts/formatter.py with format_signal_alert and get_expert_position_details
    2. Use html.escape() for all user-generated content (market questions, addresses)
    3. Truncate wallet addresses for readability: first 10 chars + "..." + last 6 chars
    4. Use Telegram HTML tags: b for headers, code for addresses/numbers, a href for links
    5. Event type icons: NEW="New Signal", STRENGTHENING="Signal Strengthened", WEAKENING="Signal Weakened", LOST="Signal Lost"
    6. Structure: Header -> Market -> Signal Metrics -> Expert Details -> Positions -> Link
    7. Position sizes formatted with Decimal precision (no float)
    8. Fast-follower count derived from signal.follower_classifications values

    Use from html import escape for HTML safety.
    Import SignalResult from src.signals.pipeline.
    Import Position from src.db.models for get_expert_position_details.
    Follow pure function pattern for format_signal_alert (no DB access in formatter itself).
    get_expert_position_details is the DB accessor, kept separate for testability.
  </implementation>
</feature>
