<feature>
  <name>Signal Event Detection</name>
  <files>src/alerts/__init__.py, src/alerts/detector.py, tests/test_signal_detector.py</files>
  <behavior>
    Signal event detection compares the latest SignalSnapshot to the previous snapshot for the same
    (market_id, direction) pair and returns the event type string.

    The function signature:
    ```python
    def detect_signal_event(
        session: Session,
        market_id: str,
        direction: str,
    ) -> str | None
    ```

    Uses get_signal_history(session, market_id, direction=direction, limit=2) to get latest + previous.

    Event classification rules (from user CONTEXT.md locked decisions):
    - NEW: First snapshot for this market+direction is active, OR previous was inactive and latest is active
    - STRENGTHENING: Both active, confidence_score increased >= 5 points (Decimal("5"))
    - WEAKENING: Both active, confidence_score decreased >= 5 points (Decimal("-5"))
    - LOST: Previous was active, latest is inactive
    - None: No significant change (confidence delta < 5 points), or first snapshot is inactive

    Cases (input -> expected output):
    1. Single active snapshot (first ever) -> "NEW"
    2. Single inactive snapshot (first ever) -> None
    3. inactive -> active -> "NEW" (re-emergence)
    4. active -> inactive -> "LOST"
    5. active(60) -> active(66) -> "STRENGTHENING" (delta=6, >= 5)
    6. active(60) -> active(53) -> "WEAKENING" (delta=-7, >= -5)
    7. active(60) -> active(63) -> None (delta=3, < 5, noise)
    8. active(60) -> active(57) -> None (delta=-3, > -5, noise)
    9. active(60) -> active(65) -> "STRENGTHENING" (delta=5, exactly threshold)
    10. active(60) -> active(55) -> "WEAKENING" (delta=-5, exactly threshold)
    11. No history at all (empty list) -> None
    12. inactive -> inactive -> None (no change)
  </behavior>
  <implementation>
    1. Create src/alerts/__init__.py with conditional import for detector module
    2. Create src/alerts/detector.py with detect_signal_event function
    3. Import get_signal_history from src.signals.queries
    4. Use Decimal("5") as CONFIDENCE_CHANGE_THRESHOLD constant
    5. Compare latest and previous snapshot status and confidence_score fields
    6. Return event type string or None

    Use loguru logger for debug logging of event detection decisions.
    Follow pure function pattern - session is only used to call get_signal_history.
  </implementation>
</feature>
