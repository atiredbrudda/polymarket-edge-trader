<feature>
  <name>CLI Formatters and Commands</name>
  <files>pyproject.toml, src/cli/__init__.py, src/cli/formatters.py, src/cli/commands.py, tests/test_formatters.py, tests/test_cli.py</files>
  <behavior>
    Formatter functions (src/cli/formatters.py):
    All formatters are pure functions: data in → Rich renderable out. No database access, no side effects.

    1. truncate_address(address: str) -> str
       - "0xAbCdEf1234567890AbCdEf1234567890AbCdEf12" → "0xAbCd...Ef12"
       - Returns full address if fewer than 10 chars

    2. format_markets_table(markets: list) -> Table
       - Columns: Market (question), Game (taxonomy slug), Status (active/resolved)
       - Filters: Show only markets with classification (ESports)
       - Input: list of dicts or objects with question, slug, active fields

    3. format_trader_profile(trader_address: str, summaries: list, positions: list, scores: list) -> Group
       - Section 1: Header panel with address
       - Section 2: Category summary table (category, volume, trade count)
       - Section 3: Current positions table (market, direction, size, entry price)
       - Section 4: Expertise scores table (game, score, percentile, specialization)

    4. format_signals_table(signals: list) -> Table
       - Columns: Market, Direction, Confidence, Experts, First Mover Address
       - Confidence as percentage with color hint
       - Function: truncate_address used on First Mover column

    5. format_leaderboard_table(entries: list, game_slug: str) -> Table
       - Columns: Rank, Trader, Score, Win Rate %
       - Hidden: PnL columns (per user decision)
       - Input: list of LeaderboardEntry dataclasses

    6. format_sweep_summary(results: dict) -> Panel
       - Simple summary panel: "Sweep Complete. processing_time=X.Xs"
       - Stats: markets_count, signals_count, alerts_sent

    Click Commands (src/cli/commands.py):

    1. Address Lookup:
       - Implement partial matching helper: find_trader_by_prefix(session, partial_address)
       - Normalize input: lowercase, strip whitespace
       - Add 0x prefix if missing
       - Match against trader_address using LIKE query: address LIKE '{input}%'
       - If 0 matches: Print error "No trader found matching '{partial}'"
       - If 1 match: Use full address
       - If >1 match: Print error "Ambiguous address '{partial}' matches {n} traders. Provide more characters."

    2. Game Slug Handling:
       - `leaderboard` command requires game_slug argument (not optional)
       - If provided but invalid (no matching taxonomy slug):
         - Query: SELECT DISTINCT slug FROM taxonomy WHERE slug LIKE 'esports.%'
         - Print error: "Game '{slug}' not found. Available games:\n  - esports.cs2\n  - esports.lol\n  ..."
       - Valid slugs proceed to get_game_leaderboard()

    Each command:
    - Create Console() instance at command start (not shared globally)
    - Default: Print progress steps (console.status or simple print)
    - --verbose: Set loguru logger to DEBUG level via logger.remove() + logger.add(sys.stderr, level="DEBUG")
    - After verbose setup, all pipeline functions using loguru will output debug logs
  </behavior>
  <implementation>

    7. Create tests/test_cli.py with CliRunner tests for help text and command registration
    8. Run `pip install -e .` to register entry point

    Formatters use Rich Console for output. Commands create a Console() and print() the renderable.
    Use loguru for debug-level logging in commands.
  </implementation>
</feature>
