  <name>Consistency Detection</name>
  <files>src/evaluation/consistency.py, tests/test_consistency.py</files>
  <behavior>
    Pure functions for consistency analysis. Imports from metrics.py, timeframes.py, and profiles.py.

    1. ConsistencyResult (frozen dataclass):
       - is_consistent: bool
       - consistency_score: Decimal  (0-100, higher = more consistent)
       - primary_signal: str  ("stable", "streaky", "insufficient_data")
       - secondary_signal: str | None  ("alternating", "clustered", None)
       - timeframe_win_rates: dict[str, Decimal | None]  (win rate per window)
       - win_rate_variance: Decimal | None  (variance across windows)
       - low_confidence_windows: list[str]  (windows with < 5 resolved)
       - profile_type: str  ("selective" or "active")

    2. calculate_consistency(
         positions_by_timeframe: dict[str, list],
         profile_type: str,
         sparse_threshold: int = 5,
       ) -> ConsistencyResult

       Primary signal (cross-timeframe stability):
       - For each timeframe (30d, 90d, all-time), calculate win rate from resolved positions
       - Skip 7d window for consistency (too noisy for meaningful comparison)
       - Flag windows with < sparse_threshold resolved markets as low-confidence
       - Calculate variance of win rates across qualifying windows (non-low-confidence)
       - If fewer than 2 qualifying windows: primary_signal = "insufficient_data"
       - Determine consistency bar from profile_type via get_profile_consistency_bar:
         - selective: variance < 100 -> "stable"
         - active: variance < 50 -> "stable"
         - Otherwise: "streaky"

       Consistency score calculation:
       - Base: 100 - (variance * weighting_factor)
       - Clamp to [0, 100]
       - If insufficient_data: score = Decimal("0")

       Cases:
       - Win rates [65%, 68%, 70%] across 30d/90d/all -> low variance -> "stable"
       - Win rates [90%, 45%, 60%] across 30d/90d/all -> high variance -> "streaky"
       - Only 1 qualifying window -> "insufficient_data"
       - All windows low-confidence -> "insufficient_data"

    3. analyze_streaks(outcomes: list[str]) -> dict
       Secondary signal (streak length analysis):
       - Input: chronologically ordered list of outcomes ("win", "loss")
       - Exclude "void" and "flat" outcomes
       - Calculate:
         - max_win_streak: int
         - max_loss_streak: int
         - avg_streak_length: Decimal
         - alternation_rate: Decimal  (transitions between W/L / total)
       - Higher alternation_rate = more consistent (70% at alternating better than clustered)
       - Return {"max_win_streak": int, "max_loss_streak": int,
                 "avg_streak_length": Decimal, "alternation_rate": Decimal,
                 "signal": str}
       - signal: "alternating" if alternation_rate >= 0.4, "clustered" otherwise

       Cases:
       - [W, L, W, L, W, L, W] -> alternation_rate = 1.0 -> "alternating"
       - [W, W, W, W, L, L, L, L] -> alternation_rate ~= 0.14 -> "clustered"
       - [W, W, L, W, L, W, W] -> alternation_rate ~= 0.67 -> "alternating"
       - Empty or single outcome -> signal = "alternating" (no streak evidence)
  </behavior>
