<feature>
  <name>Timeframe Windows</name>
  <files>src/evaluation/timeframes.py, tests/test_timeframes.py</files>
  <behavior>
    Pure functions for time-based filtering. All timestamps are UTC, timezone-naive (per existing codebase pattern).

    1. TIMEFRAME_WINDOWS = {"7d": timedelta(days=7), "30d": timedelta(days=30), "90d": timedelta(days=90), "all": None}

    2. get_timeframe_bounds(window_key: str, now: datetime | None = None) -> tuple[datetime | None, datetime]
       - Returns (start, end) where end = now (defaults to utcnow)
       - "7d" -> (now - 7 days, now)
       - "30d" -> (now - 30 days, now)
       - "90d" -> (now - 90 days, now)
       - "all" -> (None, now) — None start means no lower bound
       - Raises ValueError for unknown window_key
       - now parameter enables deterministic testing

    3. filter_positions_by_window(positions: list, window_key: str, now: datetime | None = None) -> list
       - Duck-typed positions with last_trade_timestamp attribute
       - Uses get_timeframe_bounds to get bounds
       - Returns positions where last_trade_timestamp falls within [start, end]
       - "all" returns all positions (no filtering)
       - Cases:
         - Empty list -> []
         - "7d" with positions at day 3 and day 10 -> only day 3 position
         - "all" -> all positions returned
         - Position with last_trade_timestamp == None -> excluded from time-filtered windows, included in "all"

    4. get_all_timeframe_snapshots(positions: list, now: datetime | None = None) -> dict[str, list]
       - Returns {"7d": [...], "30d": [...], "90d": [...], "all": [...]} with filtered positions per window
       - Convenience function for generating all timeframe views at once
  </behavior>
  <implementation>
    Create src/evaluation/timeframes.py with pure functions.
    All datetime operations use timezone-naive UTC (matching existing codebase).
    Accept `now` parameter for deterministic testing.
  </implementation>
</feature>

<feature>
  <name>Trader Profile Classification</name>
  <files>src/evaluation/profiles.py, tests/test_profiles.py</files>
  <behavior>
    Pure functions for trader profile classification.

    1. TraderProfile (frozen dataclass):
       - trader_address: str
       - profile_type: str  ("selective" or "active")
       - unique_markets: int
       - total_trades: int
       - threshold_used: int

    2. classify_trader_profile(positions: list, trader_address: str, threshold: int = 10) -> TraderProfile
       - Count unique market_id values across positions (unique markets entered)
       - If unique_markets < threshold -> "selective"
       - If unique_markets >= threshold -> "active"
       - Duck-typed positions with market_id attribute
       - Cases:
         - 2 positions, 2 unique markets, threshold=10 -> "selective"
         - 15 positions, 15 unique markets, threshold=10 -> "active"
         - 10 positions, 10 unique markets, threshold=10 -> "active" (at boundary)
         - 50 trades on 3 markets -> "selective" (many trades, few markets = selective per user decision)
         - Empty positions -> "selective" (0 unique markets)

    3. get_profile_consistency_bar(profile_type: str) -> dict
       - Returns consistency thresholds per profile
       - "selective": {"min_timeframes": 2, "max_variance": 100} (need stability across fewer windows)
       - "active": {"min_timeframes": 2, "max_variance": 50} (tighter bar per user decision)
       - These thresholds are starting points, tuned via validation framework
  </behavior>
  <implementation>
    Create src/evaluation/profiles.py.
    Threshold of 10 unique markets per research recommendation (Claude's discretion).
    Different consistency bars per profile per user decision:
    - Selective traders need stability across fewer windows than active traders
    - Implemented as variance thresholds that Plan 03 (consistency) will use
  </implementation>
</feature>
