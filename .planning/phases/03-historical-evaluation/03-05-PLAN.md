  <name>Out-of-Sample Validation Framework</name>

  <behavior>
    Pure functions for temporal validation. Uses temporal holdout (not k-fold) to avoid lookahead bias per research recommendation. All datetime operations use timezone-naive UTC matching existing codebase.

    Data structures:

    1. FoldResult (frozen dataclass):
       - fold_id: int
       - train_start: datetime
       - train_end: datetime
       - test_start: datetime
       - test_end: datetime
       - train_trader_count: int
       - test_trader_count: int
       - metric_scores: dict  (e.g., {"correlation": Decimal, "rank_accuracy": Decimal, "top_k_precision": Decimal})
       - weights_used: dict  (e.g., {"concentration": Decimal, "win_rate": Decimal, "recency": Decimal, "sample_size": Decimal})

    2. ValidationResult (frozen dataclass):
       - folds: list[FoldResult]
       - aggregate_scores: dict  (mean of metric_scores across folds)
       - weights_tested: dict
       - total_traders_evaluated: int
       - run_timestamp: datetime

    Functions:

    3. temporal_train_test_split(
         positions: list,
         split_date: datetime,
       ) -> tuple[list, list]
       - Split positions into train (last_trade_timestamp < split_date) and test (last_trade_timestamp >= split_date)
       - Duck-typed positions with last_trade_timestamp attribute
       - Ensures strict temporal ordering: no test data leaks into train
       - Cases:
         - All positions before split -> (all, [])
         - All positions after split -> ([], all)
         - Mix -> correctly partitioned
         - Position with last_trade_timestamp == split_date -> goes to test set
         - Empty list -> ([], [])

    4. walk_forward_validate(
         positions: list,
         n_folds: int = 5,
         test_window_days: int = 90,
         min_train_days: int = 90,
       ) -> list[tuple[datetime, datetime, datetime, datetime]]
       - Generate fold boundaries for walk-forward validation
       - Find data range: earliest and latest last_trade_timestamp in positions
       - Work backwards from latest timestamp:
         - Fold N: test = [latest - test_window, latest], train = [earliest, latest - test_window]
         - Fold N-1: test = [latest - 2*test_window, latest - test_window], train = [earliest, latest - 2*test_window]
         - Continue until n_folds reached OR remaining train period < min_train_days
       - Returns list of (train_start, train_end, test_start, test_end) tuples
       - Each fold has expanding training window (more data as you go forward)
       - Cases:
         - Sufficient data for 5 folds -> returns 5 fold boundaries
         - Only enough data for 3 folds -> returns 3 fold boundaries
         - Insufficient data for even 1 fold -> returns []
         - Empty positions -> returns []

    5. evaluate_scoring_weights(
         train_positions: list,
         test_positions: list,
         weights: dict,
         metric_fn: callable = None,
       ) -> dict
       - Apply weights to compute composite scores on train data
       - Evaluate prediction quality on test data
       - weights dict: {"concentration": Decimal, "win_rate": Decimal, "recency": Decimal, "sample_size": Decimal}
       - All weights must sum to Decimal("1.0") (normalized)
       - Raises ValueError if weights don't sum to 1.0 (within Decimal("0.001") tolerance)
       - Compute per-trader scores using weighted combination
       - Evaluate on test set:
         - correlation: rank correlation between train scores and test win rates
         - rank_accuracy: fraction of top-K traders by train score who remain top-K in test
         - top_k_precision: precision at K=10 (or min(10, num_traders))
       - metric_fn: optional callable for custom metric computation (enables extensibility)
       - Return {"correlation": Decimal, "rank_accuracy": Decimal, "top_k_precision": Decimal}
       - Cases:
         - Perfect predictor -> correlation close to 1.0
         - Random predictor -> correlation close to 0.0
         - Too few traders for top_k -> uses min(K, num_traders)
         - Empty test set -> all metrics = Decimal("0")

    6. run_validation(
         positions: list,
         weights: dict,
         n_folds: int = 5,
         test_window_days: int = 90,
         min_train_days: int = 90,
         metric_fn: callable = None,
       ) -> ValidationResult
       - Orchestrator function: generates folds, evaluates weights, aggregates results
       - Calls walk_forward_validate to get fold boundaries
       - For each fold: temporal_train_test_split -> evaluate_scoring_weights
       - Aggregates FoldResults into ValidationResult
       - Re-runnable: calling with same inputs produces same outputs (deterministic)
       - Cases:
         - Normal run -> ValidationResult with fold metrics
         - No valid folds -> ValidationResult with empty folds list
  </behavior>
