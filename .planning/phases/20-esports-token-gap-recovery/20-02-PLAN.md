Replacement approach (events-based lookup, batch-cached):
```python
# FIXED — replace the entire block above with this:
# Populate tokens for catalog-path markets (created without token data)
if catalog_condition_ids:
    needs_tokens = [
        cid for cid in catalog_condition_ids
        if session.query(Market).filter_by(condition_id=cid).first() is not None
        and session.query(Market).filter_by(condition_id=cid).first().tokens is None
    ]
    if needs_tokens:
        logger.info(f"Fetching tokens for {len(needs_tokens)} catalog-path markets via events index")
        try:
            from src.catalog.recovery import _fetch_esports_events_index
            events_index = _fetch_esports_events_index()
            populated = 0
            for cid in needs_tokens:
                tokens = events_index.get(cid)
                if tokens:
                    market = session.query(Market).filter_by(condition_id=cid).first()
                    if market and market.tokens is None:
                        market.tokens = json.dumps(tokens)
                        populated += 1
                        logger.debug(f"Populated tokens for {cid[:8]}... ({len(tokens)} tokens)")
                else:
                    logger.debug(f"No events index entry for catalog-path market {cid[:8]}...")
            try:
                session.commit()
            except Exception:
                session.rollback()
            logger.info(f"Populated tokens for {populated}/{len(needs_tokens)} catalog-path markets")
        except Exception as e:
            logger.warning(f"Events-based token population failed: {e}")
```

Key differences:
- Remove `and self.gamma_client` guard — events index does not require gamma_client
- Fetch events index ONCE for the whole batch (not once per condition_id)
- Import `_fetch_esports_events_index` from `src.catalog.recovery` (plan 01 output)
- No batch loop — events index is an in-memory dict, iteration is O(1) per cid
- Remove `time.sleep(0.05)` inner loop — only one API call (the paginated events fetch inside _fetch_esports_events_index)
- Wrap entire block in try/except to avoid breaking backfill if events API fails
- Token format stays identical: `json.dumps(tokens)` where tokens is already `[{"token_id": tid, "outcome": ""}]`

Note on Pitfall 5 (double-fetching): `ingest_trader_history_jbecker()` is called once per trader. `catalog_condition_ids` is populated for all markets encountered during that trader's backfill. The events index is fetched once inside this block (for all of that trader's catalog-path markets in one shot). The existing caller of `ingest_trader_history_jbecker()` does NOT need to be changed — the block is already per-trader, not per-trade.
