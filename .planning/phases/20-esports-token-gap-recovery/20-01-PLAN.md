**`recover_esports_token_gaps(session: Session) -> dict[str, int]`**
- Returns `{"found": int, "populated": int, "already_done": int, "patched": int, "local": int, "api": int, "fallback": int}`
- Step 1: Query gap markets — `SELECT DISTINCT m.condition_id FROM markets m JOIN trades t ON t.market_id = m.condition_id WHERE m.tokens IS NULL AND LOWER(m.category) = 'esports'` using `session.execute(text(...)).scalars().all()`
- If no gap markets, log and return immediately
- Step 2: Call `_fetch_esports_events_index()` to build the lookup
- Step 3: For each condition_id in gap_markets — look up in events_index; if found and market.tokens is None, set `market.tokens = json.dumps(tokens)`; track `populated` and `already_done`
- Step 4: `session.commit()`
- Step 5: Import and call `patch_missing_catalog_entries(session, gamma_client=None)` from `src.catalog.patcher`; merge its stats dict into the return dict
- Use loguru `logger` for structured logging (RECOVER-CATALOG prefix)
- Imports needed: `json`, `httpx`, `loguru.logger`, `sqlalchemy.orm.Session`, `sqlalchemy.text`, `src.db.models.Market`

**Token format is CRITICAL:** Must be `[{"token_id": tid, "outcome": ""}, ...]` (list of dicts). NOT `["tid1", "tid2"]` (plain strings). Tier 1 patcher calls `token_entry.get("token_id")` — plain strings return None and silently break classification.

Do NOT call `_ensure_catalog_built()` — the catalog builder DELETEs all token_catalog rows on every call and would wipe freshly-recovered rows.

```python
@cli.command("recover-catalog")
@click.option("--verbose", "-v", is_flag=True, help="Enable debug logging")
def recover_catalog_cmd(verbose):
    """Populate markets.tokens for null-token eSports gap markets, then re-patch.

    Fetches eSports events from Gamma API (tag_id=64), extracts token IDs for
    null-token markets, populates markets.tokens, and runs the patcher.

    Safe to re-run -- idempotent (skips already-populated markets).

    \b
    Examples:
        polymarket recover-catalog
        polymarket recover-catalog --verbose
    """
    logger.info("RECOVER-CATALOG command started")

    if verbose:
        logger.remove()
        logger.add(sys.stderr, level="DEBUG")

    console = Console()
    session_factory, _, _, _, gamma_client = _get_dependencies()

    from src.catalog.recovery import recover_esports_token_gaps
    with get_session(session_factory) as session:
        stats = recover_esports_token_gaps(session)

    if stats["found"] == 0:
        console.print("[green]No null-token eSports gap markets found.[/green]")
    else:
        console.print(f"[bold green]Recovery complete:[/bold green]")
        console.print(f"  Gap markets found:    {stats['found']}")
        console.print(f"  Tokens populated:     {stats['populated']}")
        console.print(f"  Already done (skip):  {stats['already_done']}")
        console.print(f"  Catalog patched:      {stats.get('patched', 0)} markets")
        console.print(f"    Local (gamma_events): {stats.get('local', 0)}")
        console.print(f"    API lookup:           {stats.get('api', 0)}")
        console.print(f"    Category fallback:    {stats.get('fallback', 0)}")

    logger.info(
        f"RECOVER-CATALOG completed: found={stats['found']}, "
        f"populated={stats['populated']}, already_done={stats['already_done']}, "
        f"patched={stats.get('patched', 0)}"
    )
```

The `logger`, `sys`, `Console`, `get_session`, and `_get_dependencies` are already imported/defined in commands.py — do not re-import them at module level.
