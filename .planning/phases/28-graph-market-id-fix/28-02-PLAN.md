## Goal

Modify `ingest_trader_history_graph()` in `src/pipeline/ingest.py` to build and use a `token_to_condition` cache from `token_catalog`, matching the JBecker ingestion pattern.

## Scope

### In Scope
- Add `_ensure_catalog_built(session)` call at start of Graph ingest
- Build `catalog_token_cache` from entire `token_catalog` table (all niches)
- Pass cache to `graph_trade_to_api_response()`
- Enable categorization and market creation for Graph trades (like JBecker path)
- Update market creation and classification logic for Graph trades

### Out of Scope
- Converter changes (Plan 28-01)
- Database migration (Plan 28-03)
- Changes to other ingestion paths


### New Flow (Mirror JBecker Pattern)
```python

self._ensure_catalog_built(session)

# Step 2: Build catalog cache (all niches, not just esports)
catalog_token_cache = {
    row.token_id: (row.condition_id, row.question, row.node_path, row.market_type)
    for row in session.query(TokenCatalog).all()
}

# Step 3: Process trades with catalog lookup
for graph_trade in graph_trades:
    trade_response = graph_trade_to_api_response(
        graph_trade, trader_address, catalog_token_cache
    )
    
    # If token in catalog, create market and classification
    if trade_response.asset_id in catalog_token_cache:
        condition_id, question, node_path, market_type = catalog_token_cache[trade_response.asset_id]
        # Create Market if not exists
        # Create MarketClassification
        # Categorize trade
    
    # Store trade with real condition_id
    trade = Trade(market_id=condition_id, ...)
    session.add(trade)
```

### Key Changes

1. **Line ~1340:** Add catalog build after fetching trades
2. **Line ~1370:** Pass cache to converter
3. **Lines ~1385-1396:** Add market creation and classification logic (similar to JBecker lines 1617-1658)
4. **Categorization:** Route esports trades to esports summaries, others to appropriate categories
